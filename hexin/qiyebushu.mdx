### 企业环境评估

部署前进行全面的环境评估：

<Steps>
<Step title="网络环境评估">
    评估将 RM-01 连接到企业内部网络的可行性：
    
    - 评估网络拓扑和安全要求
    - 识别防火墙和代理配置
    - 规划 IP 地址分配和网络分段
    - 测试网络带宽和延迟要求
    
    <Check>
    记录网络需求并获得 IT 安全团队的必要批准。
    </Check>
  </Step>

<Step title="电源供应评估">
    确保部署位置的稳定电源供应：
    
    - 验证电源插座的可用性和规格
    - 检查不间断电源（UPS）需求
    - 评估功耗对设施的影响
    - 如需要，规划电源冗余
    
    <Info>
    RM-01 最大功耗为 100W，TDP 为 60W，可实现最佳性能。
    </Info>
  </Step>
  
<Step title="物理安全审查">
    评估设备放置的物理安全性：
    
    - 评估物理访问控制
    - 审查环境条件（温度、湿度）
    - 规划设备安装和线缆管理
    - 评估防盗措施
    
    <Warning>
    确保部署位置符合企业安全标准和环境要求。
    </Warning>
</Step>

<Step title="IT 基础设施集成">
    了解现有 IT 基础设施：
    
    - 审查当前的 AI/ML 基础设施
    - 评估与现有系统的集成
    - 规划用户访问管理
    - 评估监控和日志记录要求
  </Step>
</Steps>

## 硬件部署流程

### 设备安装

<Steps>
<Step title="场地准备">
    准备安装位置：
    
    1. 选择最佳放置位置（桌面或专用机柜）
    2. 确保设备周围有充足的通风
    3. 验证电源和网络连接
    4. 准备线缆管理解决方案
    
    <Warning>
      确保设备放置在通风良好的环境中，避免阻塞散热孔。
    </Warning>
  </Step>
  
<Step title="硬件设置">
    安装 RM-01 设备：
    
    1. 插入预装模型和应用的 CFexpress 存储卡
    2. 插入用于系统存储的 MicroSD 卡
    3. 连接 USB-C 电源适配器
    4. 如需有线网络，连接 USB-C 转以太网适配器
    
    <Note>
    小心处理存储卡，避免损坏连接器和数据损坏。
    </Note>
</Step>
</Steps>

### 系统初始化

<Steps>
<Step title="初始启动">
    开机并初始化系统：
    
    1. 按下电源按钮启动设备
    2. 通过连接的笔记本电脑（USB-C）监控初始化过程
    3. 等待自动加载模型和应用（约 5 分钟）
    4. 验证系统达到就绪状态
    
    <Info>
    在初始化期间，您可以通过 USB-C 端口连接笔记本电脑来监控设备状态并在需要时进行故障排除。
    </Info>
  </Step>
  
<Step title="系统验证">
    确认初始化成功：
    
    - 检查系统状态指示器
    - 验证模型加载完成
    - 测试基本设备响应性
    - 验证存储卡识别
    
    <Check>
    系统应在初始化约 5 分钟后显示就绪状态。
    </Check>
  </Step>
</Steps>

### 网络配置

根据企业要求配置网络连接：

<Tabs>
<Tab title="有线网络">
    配置以太网连接：
    
    ```bash
    # 静态 IP 配置
    sudo ip addr add 192.168.1.100/24 dev eth0
    sudo ip route add default via 192.168.1.1
    
    # DNS 配置
    echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
    ```
    
    <Check>
    通过 ping 测试外部服务器验证网络连接。
    </Check>
</Tab>

<Tab title="DHCP 配置">
    启用自动 IP 分配：
    
    ```bash
    # 启用 DHCP 客户端
    sudo dhclient eth0
    
    # 验证 IP 分配
    ip addr show eth0
    ```
    
    <Note>
    在大多数企业环境中，建议使用 DHCP 以简化网络管理。
    </Note>
</Tab>
</Tabs>
